회원가입 약관 동의 플로우차트

마카다미아 멤버스 CRM — 가입 경로별 약관 동의 흐름 (하이브리드 관리 모델)

v1.0 · 2026-03-12

시나리오 1: 대교 단독 가입

사용자가 대교 직접 서비스에 가입하는 경우. 그룹사 약관 없이 대교 약관만 동의합니다.

대교 약관만 적용
👤 사용자 — 대교 서비스 가입 시작
📱 PASS 본인인증
이름, 생년월일, 휴대폰번호 수집 및 본인확인
이름 + 생년월일 + 휴대폰번호로 통합회원 DB 조회
🆕 조회 결과: 신규 회원
3가지 정보 일치하는 통합회원 없음 → 전체 약관 동의 필요
📋 대교 약관/동의서 STEP 1
☑ 이용약관 서비스 필수
☑ 개인정보 수집·이용 동의 서비스 필수
☐ 마카다미아 마케팅 수신 동의 공통 선택
회원 정보 입력
이름, 이메일, 비밀번호 등 입력
✅ 대교 회원 가입 완료

시나리오 2: 그룹사(에듀캠프) 트니트니 최초 가입

사용자가 에듀캠프의 트니트니을 통해 최초 통합회원 가입하는 경우. 대교 약관 + 그룹사 약관 모두 동의합니다.

대교 약관 그룹사 공통 약관 트니트니 약관
👤 사용자 — 에듀캠프 트니트니 가입 시작
📱 PASS 본인인증
이름, 생년월일, 휴대폰번호 수집 및 본인확인
이름 + 생년월일 + 휴대폰번호로 통합회원 DB 조회
🆕 조회 결과: 신규 회원
3가지 정보 일치하는 통합회원 없음 → 대교 + 그룹사 전체 약관 동의 필요
📋 대교 약관/동의서 STEP 1
☑ 이용약관 서비스 필수
☑ 개인정보 수집·이용 동의 서비스 필수
☑ 개인정보 제3자 제공 동의 (대교→에듀캠프) ★ 그룹사 가입시만, 서비스별 서비스 필수
☐ 마케팅 목적 제3자 제공 동의 (대교→에듀캠프) ★ 그룹사 가입시만 공통 선택
☐ 마카다미아 마케팅 수신 동의 공통 선택
📋 에듀캠프 약관/동의서 STEP 2
☑ 이용약관 (트니트니) 트니트니 필수
☑ 개인정보 제3자 제공 동의 (에듀캠프→대교) ★ 대교 연동 필수, 서비스별 트니트니 필수
☐ 마케팅 목적 제3자 제공 동의 (에듀캠프→대교) 공통 선택
회원 정보 입력
이름, 이메일, 비밀번호 등 입력
✅ 에듀캠프 트니트니 통합회원 가입 완료

💡 핵심 포인트

  • 서비스 단위(이용약관, 개인정보 제3자 제공 동의): 트니트니 전용으로 동의 수집 → service_id = 트니트니
  • 공통(그룹사 단위)(마케팅 목적 제3자 제공 동의): 에듀캠프 그룹사 전체에 적용 → service_id = NULL
  • 대교 약관과 그룹사 약관이 순차적으로 노출됨

시나리오 3: 이미 트니트니 가입 회원 → 오감올리 추가 가입

에듀캠프 트니트니로 이미 통합회원 가입된 사용자가 동일 그룹사의 오감올리를 추가 이용하려는 경우.

그룹사 공통 → 이미 동의됨 오감올리 약관 → 신규 동의
👤 사용자 — 에듀캠프 오감올리 가입 시작
📱 PASS 본인인증
이름, 생년월일, 휴대폰번호 수집 및 본인확인
이름 + 생년월일 + 휴대폰번호로 통합회원 DB 조회
🔍 조회 결과: 기존 회원 특정
3가지 정보 일치하는 통합회원 발견
→ 기존 아이디: user123@email.com
→ 가입 서비스: 에듀캠프 트니트니
특정된 회원의 기동의 이력 조회
⚡ 기동의 판단 기준
판단 시점: 회원이 특정된 직후 (PASS 인증 → 회원 조회 완료 후)
판단 기준: 동일 그룹사(에듀캠프)의 공통 약관(service_id = NULL)에 동의 이력이 있는가?
✅ 이미 동의됨
🔒 그룹사 공통 약관 SKIP
— 마케팅 목적 제3자 제공 동의 (에듀캠프→대교) 스킵
service_id = NULL → 그룹사 단위 1회 동의로 충분
🆕 신규 동의 필요
📋 오감올리 전용 약관 STEP 1
☑ 이용약관 (오감올리) 오감올리 필수
☑ 개인정보 제3자 제공 동의 (에듀캠프→대교) ★ 대교 연동 필수, 서비스별 오감올리 필수
service_id = 오감올리 → 서비스별 별도 동의
✅ 오감올리 이용 등록 완료

💡 핵심 포인트 — 하이브리드 모델의 핵심 시나리오

  • 공통(그룹사 단위) 약관은 동일 그룹사 내에서 1회 동의로 충분 → 오감올리 가입 시 재수집 안 함
  • 서비스 단위 약관은 서비스별로 이용 조건·수집 항목이 다르므로 오감올리 전용 동의를 별도 수집
  • DB에서 service_id IS NULL이면 공통, service_id = 오감올리이면 서비스 단위로 구분
  • 사용자 UX: 이미 동의한 공통 약관은 노출하지 않고, 오감올리 전용 약관만 간소화하여 노출

📌 하이브리드 관리 모델 핵심 원칙

  • 공통(그룹사 단위): 마케팅 목적 제3자 제공 동의 등 → 법인이 수신 주체이므로 그룹사 1회 동의로 충분
  • 서비스 단위: 이용약관·개인정보 제3자 제공 동의 → 서비스별 이용 조건·제공 항목이 다르므로 서비스별 별도 동의
  • DB 구분: term_items.service_id IS NULL → 공통 / service_id = 값 → 서비스 단위
  • UX 최적화: 서비스 추가 시 이미 동의한 공통 약관은 미노출 → 간소화된 동의 화면 제공
