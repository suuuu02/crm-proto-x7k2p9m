OAuth 클라이언트 등록

새로운 OAuth 2.0 클라이언트를 등록합니다. 등록 완료 시 Client ID와 Secret이 발급됩니다.

기본 정보

클라이언트가 속할 서비스를 선택합니다.

클라이언트를 구분하기 위한 이름입니다.

기밀 클라이언트: Client Secret을 안전하게 보관할 수 있는 환경 / 공개 클라이언트: SPA, 모바일 앱 등

OAuth 설정

인증 완료 후 리다이렉트될 URI (Authorization Code Flow 사용 시 필수)

로그아웃 완료 후 리다이렉트될 URI (Authorization Code Flow 사용 시 필수)

Authorization Code Flow 재시작 엔드포인트 URI (Authorization Code Flow 사용 시 필수)

Front Channel 로그아웃 요청 URI (서비스의 SSO 연동이 '사용'인 경우 필수)

⚠️ 선택한 서비스는 SSO 연동이 활성화되어 있습니다. Front Channel Logout URI는 필수 입력 항목입니다.

OAuth 2.0 인증 흐름

  • Authorization Code: 서버 사이드 웹 애플리케이션에 권장 (PKCE 지원)
  • Implicit: SPA에서 사용 (보안상 Authorization Code + PKCE 권장)
  • Client Credentials: 서비스 간 통신 (M2M)에 사용

사용 설정

비활성 시 해당 클라이언트로의 인증 요청이 거부됩니다.

클라이언트 등록 안내

  • Client ID: 등록 완료 시 자동으로 생성됩니다. 공개되어도 괜찮은 식별자입니다.
  • Client Secret: 등록 완료 시 한 번만 표시됩니다. 반드시 안전한 곳에 저장하세요.
  • Redirect URI: 정확히 일치하는 URI로만 리다이렉트가 가능합니다. 와일드카드는 지원하지 않습니다.
